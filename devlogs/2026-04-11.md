### Work Completed

**`src/models/ctc_head.py` (281 lines):**

- `CharTokenizer`: 29-token vocabulary. Blank reserved at index 0 to match `torch.nn.CTCLoss` convention. `encode("hello")` → `[9, 6, 13, 13, 16]`. `decode(...)` skips blanks. Round-trip is exact.
- `CTCHead`: dropout → `nn.Linear(encoder_dim, vocab_size)` → `log_softmax` over the vocab axis. Intentionally minimal — all temporal modeling happens in the upstream TCN.
- `CTCHeadConfig`: dataclass for `encoder_dim`, `vocab_size`, `dropout`.
- `compute_ctc_loss(log_probs, target_indices, input_lengths, blank_index)`: wraps `F.ctc_loss` with target packing. Handles the `(B, T, V) → (T, B, V)` permutation that PyTorch CTC expects, builds the flat target tensor, and uses `zero_infinity=True` to avoid NaN cascades when a target is impossible to align.
- `ctc_greedy_decode(log_probs, blank_index)`: argmax + collapse repeats + remove blanks. Returns `list[list[int]]` for the batch. Used during training-time validation.

**`src/data/gaddy_dataset.py` (358 lines):**

- `GaddyUtterance`: `__slots__` dataclass holding session_id, text, raw EMG, is_silent flag, source book.
- `GaddyDataset(Dataset)`:
  - Walks the Gaddy directory tree (`closed_vocab/voiced/`, `closed_vocab/silent/`, `open_vocab/voiced/`, `open_vocab/silent/`).
  - `mode` filter: `"voiced"`, `"silent"`, or `"both"`.
  - `vocab` filter: `"closed"`, `"open"`, or `"both"`.
  - Optional `preprocessor` and `tokenizer` injection — if provided, applied at `__getitem__`. Allows the same dataset to be used for raw-data inspection or end-to-end training.
  - Lazy preprocessor fitting — fits on the first utterance if not already fitted, so calling code does not need to remember to call `fit()` first.
  - `_normalize_text(text)`: lowercases, strips punctuation other than apostrophes, collapses whitespace. Matches the `CharTokenizer` vocabulary exactly.
